Binary Search Tree Operations

This algorithm provides various operations on a Binary Search Tree (BST):

=== TRAVERSAL OPERATIONS ===

1. In-Order Traversal (left → root → right)
   - Visits nodes in ascending order for BST
   - Time Complexity: O(n)
   - Use case: Get sorted sequence of values

2. Pre-Order Traversal (root → left → right)
   - Visits parent before children
   - Time Complexity: O(n)
   - Use case: Copy tree structure, prefix expression

3. Post-Order Traversal (left → right → root)
   - Visits children before parent
   - Time Complexity: O(n)
   - Use case: Delete tree, postfix expression

=== MODIFICATION OPERATIONS ===

4. Search
   - Find a target value in the BST
   - Time Complexity: O(h) where h is tree height
   - Best case: O(log n) for balanced tree
   - Worst case: O(n) for skewed tree

5. Insert
   - Find the correct position to insert a new value
   - Time Complexity: O(h)
   - Maintains BST property (left < parent < right)

6. Delete
   - Locate a node to be deleted
   - Time Complexity: O(h)
   - Note: Actual deletion with rebalancing would require tree modification

=== HOW TO USE ===

Change the 'operation' parameter to select which operation to visualize:
- 'inorder', 'preorder', 'postorder' for traversals
- 'search', 'insert', 'delete' for modifications (also set 'target' value)

=== VISUALIZATION ===

- Orange (Current): The node currently being processed
- Green (Visited): Nodes that have been visited

Space Complexity: O(h) - recursive call stack where h is tree height
