INSERTION SORT - Algorithm Explanation
======================================

What is Insertion Sort?
------------------------
Insertion Sort works by building a sorted array one element at a time. It's like sorting a hand of playing cards - you pick up one card at a time and insert it into its correct position among the cards you've already sorted.

How It Works
--------------
1. Start with the second element (first element is trivially sorted)
2. Compare it with elements in the sorted portion
3. Shift larger elements one position to the right
4. Insert the current element in its correct position
5. Repeat for each remaining element

Detailed Steps:
• Start: First element is considered sorted
• Pass 1: Take second element, compare with first, insert in correct position
• Pass 2: Take third element, insert in correct position among first two
• Pass 3+: Continue, inserting each new element into its correct position
• Done when all elements have been inserted

Key Concepts
--------------
• Comparison-Based: Compares elements to find insertion position
• In-Place: Doesn't require extra memory for another array
• Stable: Equal elements maintain their original relative order
• Adaptive: Performance improves on nearly sorted data
• Online: Can sort data as it receives it, without needing all data upfront

Time Complexity
-------------------
• Best Case: O(n) - Array already sorted
• Average Case: O(n²) - Random order
• Worst Case: O(n²) - Array in reverse order
• Space: O(1) - In-place sorting

Advantages
-----------
• Efficient on small datasets and nearly sorted data
• Stable - maintains relative order of equal elements
• Online - can process data as it arrives
• Fewer comparisons than bubble sort on average
• Very efficient for linked lists

Disadvantages
--------------
• Still O(n²) in worst case
• Requires shifting elements (can be costly)
• Not as efficient as Quick Sort or Merge Sort for large arrays

When to Use Insertion Sort?
----------------------------
Good for:
• Small datasets (< 50 elements)
• Nearly sorted data (can be very efficient)
• Online sorting (processing data as it arrives)
• Sorting linked lists (no random access needed)
• Hybrid algorithms (used as base case in hybrid sorts like Timsort)

Avoid for:
• Large unsorted datasets
• When performance is critical
• Random-access not available (depends on array structure)

Learning Value
------------------
Insertion sort teaches the concept of building a sorted structure incrementally, and how algorithms can adapt to take advantage of partially sorted data. It's the basis for more advanced sorting techniques like Timsort.
